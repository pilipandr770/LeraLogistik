# Roadmap

Послідовність фаз від MVP до повноцінного продукту. Фази не повинні виконуватись паралельно — досвід показує, що додавати наступний агент до системи, де попередній ще не відпрацював 100 реальних угод, це прямий шлях до хаосу.

---

## Фаза 0 — Підготовка (зараз)

- [x] Структура проекту, ORM, адаптер Lardi, Matcher, дашборд
- [ ] Активувати Lardi API token через їх підтримку
- [ ] Створити репо на GitHub
- [ ] Задеплоїти на Render
- [ ] Провести smoke-test: health check Lardi → ingest → matcher

**Критерій виходу:** дашборд у продакшені показує реальні українські вантажі і матчі.

---

## Фаза 1 — Ручні операції + спостереження (тиждень 1-2)

Мета: зібрати якомога більше даних про те, як насправді працює українська логістика, не роблячи нічого автоматичного.

- [ ] Lera щодня переглядає ~20-50 AI-матчів і ставить оцінки "добре / погано / неочевидно"
- [ ] Зберігати human feedback у новій таблиці `match_feedback` (додати міграцію)
- [ ] Записувати сирі телефонні розмови з перевізниками (з їхньої згоди) — джерело промптів для negotiator'а в майбутньому
- [ ] Додати сторінку deal'ів, щоб вручну фіксувати перші угоди

**Критерій виходу:** 10-20 реальних угод укладено, з них >80% пройшли через запропонований AI матч.

---

## Фаза 2 — Pricing agent (тиждень 3-4)

- [ ] Таблиця `price_samples` з ринковими цінами (витягуємо з Lardi payment)
- [ ] Агент пропонує ціну для нового вантажу на основі:
  - середньої ціни на схожий маршрут за останні 30 днів
  - сезонного коефіцієнта
  - типу кузова
  - відстані deadhead вибраної машини
- [ ] На дашборді біля кожного матчу показуємо "рекомендована ціна: X UAH"

**Критерій виходу:** ціни, запропоновані AI, в межах ±15% від реальної ціни угоди у 70% випадків.

---

## Фаза 3 — Email parsing (тиждень 5-6)

Не всі клієнти ходять на Lardi. Багато відправляють заявки на email вільною формою.

- [ ] Підключити inbox (пропозиції: Postmark, SendGrid Inbound, або свій IMAP)
- [ ] Parser-агент розбирає email → структурований Load
- [ ] Додаємо у БД зі `source="email"`
- [ ] Автовідповідь "ми отримали вашу заявку, відповімо протягом години"

**Критерій виходу:** 100% email-заявок парсяться без втрати критичних полів (маршрут, вага, дата).

---

## Фаза 4 — Telegram/Viber integrations (тиждень 7-8)

- [ ] Telegram Bot API: створення бота, обробка повідомлень перевізників
- [ ] Viber Business API (дорожче, але потрібне — у Україні водії частіше на Viber)
- [ ] Автоматично створюються Carrier при першому контакті
- [ ] Повідомлення прив'язуються до Negotiation (вручну через UI на цьому етапі)

**Критерій виходу:** Lera відповідає перевізникам через один інтерфейс замість трьох мобільних додатків.

---

## Фаза 5 — Negotiator agent у режимі draft (тиждень 9-12)

Найнебезпечніший етап. Вмикаємо поступово.

- [ ] Агент готує чернетки повідомлень на основі:
  - контексту переговорів
  - запропонованої ціни
  - історії роботи з цим перевізником
- [ ] Lera бачить чернетку, може редагувати, натискає "Надіслати"
- [ ] Ніколи не вмикати `AGENT_NEGOTIATOR_AUTO_SEND=true`, поки не буде 100+ угод, проведених через draft-mode

**Критерій виходу:** 50% чернеток відправляються Lera без редагування.

---

## Фаза 6 — Risk agent (тиждень 13-14)

- [ ] Інтеграція з ЄДРПОУ (відкритий API)
- [ ] Перевірка через Youcontrol або Opendatabot (платні)
- [ ] Скрейпинг рейтингу з Lardi (через адаптер, не браузерну автоматизацію)
- [ ] При створенні нового Carrier'а → автоматична оцінка trust_score

**Критерій виходу:** жоден deal не укладається з перевізником з trust_score < 40.

---

## Фаза 7 — Document agent (тиждень 15-16)

- [ ] Генерація CMR у PDF (українська + англійська версії)
- [ ] Генерація договору експедирування з шаблонної .docx
- [ ] Генерація рахунку для експедиторської винагороди
- [ ] Інтеграція з Monobank API для виставлення рахунків

**Критерій виходу:** після укладення угоди Lera натискає одну кнопку, отримує PDF/DOCX, відправляє клієнту.

---

## Фаза 8 — Перший скейл (місяць 5-6)

- [ ] Додати адаптер Della (`app/adapters/della.py`) — другий ринок
- [ ] APScheduler для автоматичного щохвилинного ingest з обох бірж
- [ ] Можливо: найняти одного диспетчера-асистента (part-time) для того, щоб Lera могла сфокусуватись на прямих клієнтах

**Критерій виходу:** 100+ угод на місяць, позитивний операційний кеш-флоу.

---

## Фаза 9 — Міжнародна експансія (місяць 7-12)

Це вже коли український ринок відпрацьований.

- [ ] Україна → Польща маршрути (вже на Lardi)
- [ ] Реєстрація на Trans.eu (€129/міс)
- [ ] Адаптер Trans.eu — їх API вимагає оплаченої підписки і окремої угоди на API-доступ
- [ ] Локалізація дашборду на польську, англійську
- [ ] Negotiator-промпти для кожної мови окремо (не можна один промпт для всіх)

**Критерій виходу:** >30% обороту з міжнародних маршрутів.

---

## Фаза 10 — Timocom і DACH (рік 2)

Timocom — ринок Німеччина/Австрія/Швейцарія, найдорожчий і найцінніший. Вхід вимагає:
- німецька реєстрація (GmbH або UG, разом з українським ФОП)
- підтверджена фінансова спроможність
- історія роботи на Trans.eu
- окрема угода на API (€300-800/міс)

Це вже не MVP-проект, це повноцінний бізнес з 5-10 співробітниками + AI.
